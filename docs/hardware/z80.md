# Zilog Z80 (CPC Z80A) — implementation reference

The CPC's CPU is an **NMOS Z80A at 4 MHz**. This guide is the hardware reference
for building it as a cycle-stepped `Device` (see `docs/hw-spec.md`): the register
model, the machine-cycle structure, the decode algorithm, the *exact* flag and
MEMPTR/Q behaviour (where correctness is actually won), interrupts, and the CPC
integration. It is written to be reimplementable in any language.

**Variant matters.** The CPC uses an **NMOS** Z80 (not CMOS). Two behaviours
differ between variants — the `SCF`/`CCF` undocumented flags and the `OUT (C),0`
operand. Everything here describes **NMOS**, which is what FUSE and jsmoo encode.

**Authoritative sources** (build from these, not from legacy `z80.cpp`):
Zilog Z80 CPU User Manual (UM0080); Sean Young, *The Undocumented Z80 Documented*
(v0.91); Patrik Rak, *MEMPTR* and *The Q register* (2018); the FUSE test suite
(`tests.in`/`tests.expected`) and SingleStepTests/jsmoo (cycle-exact); the z80.info
*Decoding Z80 opcodes* algorithm. The legacy core is the CPC-integration golden
master only (its CPC instruction totals, `src/z80.cpp:889` `cc_op[]`).

## 1. Register file

| 16-bit | high / low | notes |
|---|---|---|
| `AF` | A, F | accumulator + flags |
| `BC` | B, C | |
| `DE` | D, E | |
| `HL` | H, L | |
| `AF' BC' DE' HL'` | — | alternate set (swapped by `EX AF,AF'` / `EXX`) |
| `IX` | IXH, IXL | index; addressed as `(IX+d)` |
| `IY` | IYH, IYL | index |
| `SP` | — | stack pointer |
| `PC` | — | program counter |
| `WZ` | W, Z | **MEMPTR** — internal; not directly visible but leaks via flags (§5) |
| `I` | — | interrupt vector page (IM2) |
| `R` | — | DRAM refresh counter (7-bit; bit 7 software-writable, not incremented) |

Plus the internal latches: `IFF1`, `IFF2` (interrupt enable), `IM` (0/1/2),
`halted`, and `Q` (the flags-written latch, §5).

All of these are in the device's serialized state (spec §5); `WZ` and `Q` are the
internal ones a "merely plausible" Z80 omits.

## 2. The flag register F

```
 bit:  7   6   5   4   3   2    1   0
       S   Z   YF  H   XF  P/V  N   C
```

- **S** sign = bit 7 of result. **Z** zero = result == 0. **H** half-carry (bit 3→4).
  **C** carry. **N** add/subtract (1 after a subtract). **P/V** parity (logical ops)
  or signed overflow (arithmetic).
- **YF (bit 5), XF (bit 3)** are *undocumented* — normally **copied from bits 5 and 3
  of the result**. Exceptions that don't follow "copy from result":
  - `BIT n,(HL)` / `BIT n,(IX+d)`: YF/XF come from the **high byte of WZ** (the
    address), not the tested value.
  - `BIT n,r`: YF/XF from the tested register's bits 5/3.
  - `SCF`/`CCF`: the NMOS Q-dependent rule, §5.
  - block/`CPI`/`CPD`/`INI`/etc.: special rules (see Sean Young §4–5).

The flag *result tables* (the value of each flag for `ADD`/`SUB`/`AND`/etc., the
parity table, the DAA table) must match FUSE exactly. Build the parity lookup and
the add/sub half-carry/overflow helpers once and reuse.

## 3. Machine cycles (the unit the FSM advances)

The FSM is a sequence of **machine cycles (M-cycles)**, each a few T-states. Per
the datasheet:

| M-cycle | T-states | bus activity |
|---|---|---|
| **M1** opcode fetch | 4 | T1–T2: `PC` on addr, `m1`+`mreq`+`rd`; T3–T4: refresh addr `(I<<8)|R` + `mreq`+`rfsh`, **R bumped** |
| memory read | 3 | `mreq`+`rd`; data sampled end of T2 (or T3 if WAIT) |
| memory write | 3 | `mreq`+`wr` |
| I/O read/write | 4 | `iorq`+`rd`/`wr`; **one automatic Tw** is built in |
| internal (no bus) | 1–5 | computation/delay cycles |

**CPC stretch (from `docs/hw-spec.md` §3.1, Bread80-confirmed):** each **3 T memory
read/write M-cycle becomes 4 T** — the Gate Array holds `cpu.wait` for one extra T.
M1 (already 4 T) and internal cycles are unchanged. This is why every CPC
instruction total is a multiple of 4 (e.g. `LD A,(HL)` = M1 4 + read 3→4 = **8 T**,
matching `cc_op[]`). In the pin model: while `cpu.wait` is asserted the FSM holds
its current M-cycle (does not advance), and the byte is sampled the cycle WAIT
releases.

The R refresh: `R = (R & 0x80) | ((R + 1) & 0x7F)` once per M1 (and once per prefix
byte — each `DD`/`FD`/`ED`/`CB` is its own M1, so e.g. `DD CB` bumps R twice). The
**interrupt/NMI acknowledge is itself an M1 cycle**, so R is bumped once per accepted
interrupt too.

**Modeling convention for 5 T / 6 T M1 cycles.** A few instructions extend their
opcode-fetch M1 to 5 T (`PUSH`, `RST`, `RET cc`, `CALL cc`, `DJNZ`, the interrupt
acknowledge) or 6 T (`LD SP,HL/IX/IY`). The engine keeps the M1 engine uniform at
4 T and models the extra 1 T / 2 T as a **leading `internal` cycle** before the
following M-cycle (e.g. `PUSH` = M1 4 + internal 1 + write 3 + write 3 = 11 T). The
interrupt acknowledge is dispatched from M1 T1, so its leading `internal` is one T
shorter (NMI ack = 1 + 4, INT ack = 1 + 6).

## 4. Instruction decode (algorithmic, not a 1268-entry table)

Decode the opcode byte by its octal fields (z80.info *Decoding Z80 opcodes*):

```
opcode = [ x:2 | y:3 | z:3 ]      p = y >> 1     q = y & 1
```

- **Unprefixed:** `x`/`y`/`z` select the operation family (e.g. `x=1` = `LD r,r'`
  with `z`=source, `y`=dest, and `y=z=6` ⇒ `HALT`; `x=2` = ALU `A,r`; `x=0` mixes
  16-bit ops, rotates, `DJNZ`/`JR`, etc.).
- **`CB`:** rotates/shifts (`x=0`), `BIT`/`RES`/`SET` (`x=1/2/3`), `y`=bit/op,
  `z`=register.
- **`ED`:** block ops, `IN/OUT (C)`, 16-bit `ADC/SBC`, `IM`, `LD A,I/R`, `RRD/RLD`,
  `RETI/RETN`.
- **`DD`/`FD`:** prefix selecting `IX`/`IY`; `HL`→`IX`, `(HL)`→`(IX+d)` (adds a
  displacement fetch + internal cycles), `H`/`L`→`IXH`/`IXL` for the *undocumented*
  half-register ops. A `DD`/`FD` with no index-relevant opcode is a NOP-prefix that
  still costs 4 T and bumps R.
- **`DDCB`/`FDCB`:** the displacement comes **before** the `CB` opcode byte
  (`DD CB d op`); these address `(IX+d)` and many *also* write the result back to a
  register (undocumented).

Implement decode as code over `x/y/z/p/q` plus per-prefix tables; it is ~the size
of the datasheet's tables and reads like them.

## 5. MEMPTR (WZ) and the Q register — the correctness crux

These internal facts are what separate "passes a few tests" from "passes zexall".

**Q register (Patrik Rak, 2018).** `Q` holds the value of `F` **if the last
instruction modified any flag, else 0**. Concretely: clear a `q` scratch at the
start of each instruction; whenever the instruction writes `F`, set `q = F`. At the
end, `Q = q`.

**`SCF`/`CCF` undocumented flags (NMOS).** YF/XF after `SCF`/`CCF` are:
```
F.YF |= (A.YF);  F.XF |= (A.XF);          // OR in A's bits 5/3 ...
// ... combined with the Q-dependent term: the bits set are
//     ((Q ^ F) | A) & (YF|XF)
```
The exact NMOS formula is `(F & ~(YF|XF)) | (((Q ^ F) | A) & (YF|XF))`. **Do not
hand-tune this — pin it against jsmoo/FUSE**, which encode the NMOS result; it is
the single most variant-sensitive line in the core.

**MEMPTR (WZ) update rules** (Rak, *MEMPTR*). Set `WZ` on:
- `LD A,(BC|DE)` → `WZ = rp + 1`; `LD A,(nn)` → `WZ = nn + 1`;
  `LD (nn),A` → `WZ = (A<<8) | ((nn+1) & 0xFF)`.
- `LD (nn),rp` / `LD rp,(nn)` → `WZ = nn + 1`.
- `EX (SP),HL/IX/IY` → `WZ = new HL/IX/IY`.
- `JP nn`, `CALL nn`, `RET`, `JR`, `DJNZ`, conditional taken/untaken → `WZ = target`
  (for `JR`/`DJNZ`, target even when not taken differs — see the table).
- `RST n` → `WZ = n`. `RLD`/`RRD` → `WZ = HL + 1`.
- `IN A,(n)` → `WZ = (A<<8 | n) + 1`; `OUT (n),A` → `WZ = (A<<8) | ((n+1)&0xFF)`.
- `IN/OUT (C)` → `WZ = BC ± 1`. Block `CPI`/`CPD` → `WZ ± 1`; `LDIR`/etc. → `WZ = PC+1`
  on repeat.

`WZ`'s high byte then leaks into XF/YF for `BIT n,(HL)` (§2).

## 6. Interrupts, HALT, EI delay

- **NMI** (edge-triggered, `cpu.nmi`): accepted after the current instruction;
  pushes `PC`, jumps to **0x0066**, `IFF1←0` (IFF2 preserved → `RETN` restores it).
  5 T acceptance.
- **Maskable INT** (`cpu.irq`, level): accepted between instructions **only if
  `IFF1`**. Acceptance clears `IFF1`+`IFF2`, and:
  - **IM0:** execute the opcode placed on the bus (the CPC floats `0xFF` = `RST 38h`).
  - **IM1:** `RST 0x0038` (the CPC's normal mode). 13 T.
  - **IM2:** vector = `(I<<8) | (bus byte)`; call `[vector]`. 19 T.
- **EI delay:** after `EI`, interrupts are **not** sampled until after the *following*
  instruction — modelled with the one-instruction `ei_delay` latch (CPU-internal
  state; named for the cause, `EI`, not the IFF bits). Repeated `EI`s keep deferring.
- **HALT:** executes internal NOPs (R keeps refreshing, PC frozen on HALT) until an
  interrupt; on accept, PC advances past the HALT.

**Acceptance happens only at a true instruction boundary** — `prefix == 0 && index ==
IndexReg::HL`. An interrupt is never taken between a `DD`/`FD`/`CB`/`ED` prefix and
its opcode.

**Vector byte / bus modelling (known simplification).** The acknowledge cycle is
modelled as an `internal` cycle rather than a real `iorq`+`m1` bus cycle, so the
IM0/IM2 vector byte is latched from the **committed data bus at the acceptance T1**.
On the CPC the bus rests at `0xFF` there (nothing drives it), giving the correct
`RST 38h` (IM0) and `(I<<8)|0xFF` table address (IM2). A bus that genuinely *drove*
a vector during `iorq` is **not** observed — acceptable within CPC scope (the CPC has
no vectoring peripheral and uses IM1), but a real `iorq` acknowledge cycle is the
correct fix when this core is integrated with the Gate Array.

**CPC specifics:** the GA drives `cpu.irq` (300 Hz, §3.1); on acceptance the GA does
`R52 &= 0x1F`. The data bus floats `0xFF` during INT-ack, which is why IM0/uninit
modes execute `RST 38h`. The legacy `iWSAdjust` (`src/z80.cpp:845,1681`) reduces the
ack cost by 4 T in specific prefix/16-bit-op cases — a µs-alignment artifact to
reproduce when matching the golden master's totals.

**Known undocumented-flag limitation:** the *repeating* block-I/O ops (`INIR`/`INDR`/
`OTIR`/`OTDR`) apply the single-shot `INI`/`OUTI` flag formula on every iteration. The
final (terminating) iteration — and therefore all post-instruction architectural
flags — is correct; only the mid-loop H/C/PF on a non-final iteration (observable only
by an interrupt landing mid-`OTIR`) differs from Rak's carry-propagation rule. To be
finalised against the FUSE/jsmoo undocumented suite (§8) when that oracle is wired.

## 7. Reset state

`PC=0x0000`, `I=0`, `R=0`, `IM=0`, `IFF1=IFF2=0`, interrupts disabled, not halted.
`AF` and `SP` are architecturally **undefined**; real chips and convention settle on
`0xFFFF`. General registers (BC/DE/HL/IX/IY and the alternate set) are **not**
cleared by reset. On the CPC, `/RESET` is the `cpu.reset` bus line (held ≥3 clocks).

## 8. Verification

1. **FUSE** `tests.in`/`tests.expected` — every opcode: final registers, flags,
   memory, **and per-T-state bus activity**. Run the Z80 device in isolation (no GA)
   so its raw T-state timing is checked.
2. **SingleStepTests/jsmoo z80** — 10 000 randomized cycle-exact tests per opcode;
   the authority for undocumented flags, MEMPTR, and the NMOS `SCF`/`CCF`/`Q` rule.
3. **zexall / zexdoc** — run as a CPC program through headless+IPC for whole-program
   confidence.
4. **Golden master** — CPC instruction *totals* (`cc_op[]`) once the GA's WAIT stretch
   is wired, confirming the 3 T→4 T alignment end to end.

Order of bring-up: documented opcodes → FUSE-doc + zexdoc; then undocumented +
MEMPTR + Q → zexall + jsmoo; then CPC integration → golden master.

## 9. Mapping to our Device model

- The Z80 `tick(in,out)` advances **one T-state** of the current M-cycle; it drives
  `cpu.addr`/`mreq`/`rd`/`wr`/`m1`/`rfsh` and reads `cpu.data`/`cpu.wait`/`cpu.irq`/
  `cpu.nmi`/`cpu.reset`, gated by `in.clk.cpu` (it only advances on the 4 MHz enable).
- WAIT: when `in.cpu.wait` is asserted during a memory M-cycle, hold (insert Tw).
- The micro-op sequence per instruction *is* its M-cycle list; `mread`/`mwrite`
  micro-ops auto-stall on `wait`. This is the §3.1 contract realized.
- Internal state (registers + WZ + Q + IFF/IM + halt) serializes per spec §5/§9
  (little-endian, versioned).
